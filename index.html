What is Cybersecurity?

Cybersecurity is the practice of protecting computers, servers, networks, and data from malicious attacks. Think of it as digital self-defense — just as you lock your house, you secure your systems.

🔑 CIA Triad: Confidentiality, Integrity, Availability
🔑 Threats come from hackers, malware, phishing, and human error
🔑 Anyone online needs basic cyber hygiene

What is a Microcontroller?

A microcontroller is a tiny computer on a single chip. It has a CPU, RAM, Flash storage, and programmable I/O pins. The Arduino Uno uses the ATmega328P, clocked at 16 MHz.

🔑 CPU: executes your code instruction by instruction
🔑 Flash: stores your uploaded program (32KB on Uno)
🔑 GPIO Pins: interface with LEDs, sensors, motors

Authentication verifies who you are. A weak password is like leaving your key under the doormat. Strong passwords are long, random, and unique to each account.

🔑 Minimum 12 characters — mix letters, numbers, symbols

Networks: How Devices Talk

Every internet device has an IP address — its unique identifier. Data travels in packets over TCP/IP. Understanding networking is essential for securing systems and building connected robots (IoT).

🔑 IP Address: your device's location on the network
🔑 Router: directs traffic between devices and the internet
🔑 Port: a specific "door" on a device for a service (e.g. port 80 = HTTP)

Ohm's Law & Basic Components

Before wiring anything, understand Ohm's Law: V = I × R. Key components: resistors (limit current), capacitors (store charge), LEDs (emit light), transistors (switch/amplify).

🔑 V = Voltage (V), I = Current (A), R = Resistance (Ω)
🔑 LED needs ~20mA — use 220Ω resistor with 5V supply
🔑 Color bands on resistors encode their resistance value

Encryption Deep Dive

Encryption converts plaintext into ciphertext using a key. Symmetric (AES) uses one key for both operations. Asymmetric (RSA, ECC) uses a public key to encrypt and a private key to decrypt — the backbone of HTTPS.

🔑 AES-256: current gold standard for symmetric encryption
🔑 RSA-2048: standard for asymmetric key exchange
🔑 TLS/SSL wraps all HTTPS traffic in encrypted tunnels

L08ROBOTICS

Sensors & Analog Input

Sensors convert physical phenomena into electrical signals. Arduino's 6 analog pins use a 10-bit ADC to read voltages 0–5V as values 0–1023. Common sensors: ultrasonic (distance), DHT11 (temperature), PIR (motion).

🔑 analogRead(A0) returns 0–1023 based on voltage
🔑 ADC resolution: 5V / 1024 ≈ 4.9mV per step
🔑 Use map(val, 0, 1023, 0, 100) for real-world units

SQL Injection & Web Attacks

SQL injection inserts malicious SQL code into input fields to manipulate databases. Input like ' OR '1'='1 can bypass login forms. Defense: parameterized queries and input sanitization — never trust user input.

🔑 Use prepared statements — never concatenate user input into SQL
🔑 XSS: injecting scripts into pages viewed by other users
🔑 CSRF: tricking authenticated users into unwanted actions

L10ROBOTICS

PWM & Motor Control

PWM (Pulse Width Modulation) simulates analog output by rapidly toggling a digital pin. Duty cycle controls average voltage — 50% = ~2.5V. Used for LED brightness, servo angle, and DC motor speed.

🔑 analogWrite(pin, 0–255) sets PWM duty cycle
🔑 Servo: 0° = 1ms pulse, 90° = 1.5ms, 180° = 2ms
🔑 H-Bridge (L298N) needed to reverse DC motor direction

Network Scanning & Firewalls

Network scanning (e.g. Nmap) discovers live hosts, open ports, and running services. Firewalls filter traffic by rules — blocking suspicious ports and IPs. Understanding scanning helps defenders know what attackers see.

🔑 nmap -sV 192.168.1.1 reveals service versions
🔑 Stateful firewall tracks connection state, not just ports
🔑 DMZ: isolates public-facing servers from internal network

L12ROBOTICS

I2C & SPI Communication

I2C uses 2 wires (SDA + SCL) to connect multiple devices with unique addresses. SPI uses 4 wires and is faster but needs a chip-select per device. Both are essential for connecting displays, sensors, and memory.

🔑 I2C: up to 127 devices on one bus using 7-bit addresses
🔑 SPI: MOSI, MISO, SCK, CS — full-duplex, higher speed
🔑 Wire.h handles I2C on Arduino (A4=SDA, A5=SCL)

RTOS & Real-Time Systems

An RTOS guarantees tasks execute within strict time deadlines. FreeRTOS runs on Arduino and ESP32, enabling multitasking via threads, semaphores, and message queues — essential for complex autonomous robots.

🔑 Task scheduler manages concurrent execution priorities
🔑 Semaphores prevent race conditions on shared resources
🔑 Watchdog timer resets the system if a task hangs

Reverse Engineering & Binary Exploitation

Binary exploitation abuses flaws in compiled programs. Buffer overflows overwrite stack memory to hijack execution. Tools: GDB, pwndbg, Ghidra (disassembler), pwntools (exploit scripting).

🔑 Stack layout: locals → saved RBP → return address
🔑 NX bit, ASLR, and stack canaries are modern mitigations
🔑 ROP chains string gadgets to bypass NX protection

L16ROBOTICS

PID Control Systems

PID (Proportional-Integral-Derivative) control continuously calculates an error between a desired setpoint and measured output, applying corrections. Foundation of drones, balancing robots, and CNC machines.

🔑 P term: proportional to current error — fast response
🔑 I term: accumulates past errors — eliminates steady-state error
🔑 D term: predicts future error — dampens overshoot

IoT Security & Attack Surfaces

IoT devices often run with default credentials and no encryption. Attack vectors: firmware extraction, UART debug interfaces, weak OTA updates. Security must be designed in from day one.

🔑 JTAG/UART: hardware debug ports expose firmware and shell access
🔑 Firmware analysis: binwalk, strings, Ghidra
🔑 Secure boot + code signing prevent malicious firmware

L18CYBER

Cryptographic Protocols & PKI

PKI is the system behind SSL/TLS certificates. CAs sign certificates to prove identity. Key concepts: Diffie-Hellman key exchange, ECDSA digital signatures, and X.509 certificate chains.

🔑 Diffie-Hellman: agree on a shared secret over a public channel
🔑 X.509: binds public keys to identities via CA signatures
🔑 Certificate pinning protects apps from rogue CA attacks

Pro Tips & Protocols

01
USE A PASSWORD MANAGER
Never reuse passwords. Use Bitwarden or 1Password to generate unique, complex passwords for every account.
02
ENABLE MFA EVERYWHERE
Multi-Factor Authentication adds a second layer — even if your password leaks, attackers can't get in without the second factor.
03
KEEP SYSTEMS PATCHED
Most breaches exploit known, patchable vulnerabilities. Enable auto-updates on OS, browsers, and apps.
04
ZERO TRUST MINDSET
Never assume any network or user is safe by default. Verify every request and minimize permissions.
01
ALWAYS USE A RESISTOR WITH LEDs
LEDs burn out instantly without current-limiting resistors. Calculate: R = (Vs - Vf) / If.
02
PROTOTYPE ON BREADBOARD FIRST
Before soldering, test on breadboard. It's reusable and forgiving — catches mistakes early.
03
SERIAL MONITOR IS YOUR BEST FRIEND
Use Serial.println() to debug code. It's like console.log() for hardware.
04
MIND YOUR POWER RATINGS
Arduino's 5V pin supplies ~500mA via USB. Motors and servos need external power supplies.
01
READ DATASHEETS
Every component has a datasheet. It tells you pins, voltage ranges, and protocols. Never guess.
02
BUILD SMALL, ITERATE FAST
Test one component at a time. Both security and robotics reward modular, incremental building.
03
DOCUMENT EVERYTHING
Write comments, label wires, keep a README. Future-you will thank present-you every time.
04
JOIN COMMUNITIES
Arduino forums, r/arduino, Hack The Box, TryHackMe are goldmines for hands-on learning.
